# UI Commander

我是一项用来“看见前端问题”的 skill。

有些 bug 不适合写成一段干巴巴的文字。它们发生在鼠标悬停的那一瞬间，藏在某个弹层的出现与消失之间，也可能只在你边操作边解释时，才真正说得清楚。UI Commander 的存在，就是为了接住这种时刻。

我不要求你把问题先整理成一份完整报告。你只需要像平时一样打开真实页面，动手复现，顺口讲讲“本来应该怎样，结果却发生了什么”。剩下的，我会把这段过程整理成一份能继续交给 coding agent 理解、分析、甚至直接修复的会话材料。

## 我适合什么时候出场

当你碰到下面这些场景时，可以直接叫我:

- 你知道 bug 在哪儿，但很难靠打字把过程描述清楚
- 你想把真实浏览器里的操作过程交给 agent 接着分析
- 你已经录过一次 UI Commander 会话，现在想基于那次 session 继续往下做
- 你不想先写一大段上下文，只想“让我演给你看”

如果问题更像纯逻辑、纯后端、纯接口，或者根本不需要浏览器操作来说明，那我通常就不是最合适的那位。

## 你可以怎么使用我

最常见的方式其实很自然: 直接在对话里点名我。

例如:

- `使用 ui commander 复现一个前端 bug`
- `用 ui-commander 看一下这个页面问题`
- `启动 ui commander，我来录一下`
- `使用 ui commander 分析并直接修复 http://127.0.0.1:47321/sessions/<session-id>/live`

如果你给我的是一条已有的 session URL，或者直接给我一个 session id，我会跳过重新录制，直接从那次会话继续。

如果你只是说“我这里有个前端 bug，想直接演示”，我也应该被自动触发。因为这正是我擅长的事情。

## 我们会怎么配合

UI Commander 的使用体验，应该像一次默契的接力。

### 第一种: 你现场录一遍

这是最适合第一次使用的方式。

1. 你在对话里明确要用 UI Commander
2. agent 会先进入等待状态，准备接住你即将录下的新会话
3. 你切到 Chrome，在真实页面里开始复现
4. 一边操作，一边说出你的预期和实际结果
5. 录制结束后，agent 会在当前线程里继续读这次会话，并接着分析问题

这条路最舒服的地方在于，你不需要先把问题“写对”，你只需要把问题“做出来”。

### 第二种: 你已经有一条 session

如果你手上已经有一条 UI Commander 的 localhost review 链接，或者一个现成的 session id，那就更简单。

你可以直接把它贴进对话里，让 agent 从这次录制继续向下读。这样很适合:

- 你昨天已经录过一遍，今天想继续修
- 你想把自己的复现过程转交给另一个 agent
- 你不想重新操作一次，只想基于现有工件继续推进

## 当你开始录制时，什么样的复现最有帮助

你不用“专业叙述”，自然一点反而更好。

通常来说，下面这种方式最容易让我和后续 agent 理解你的意图:

- 先点出你正在操作哪一块区域
- 说清楚你本来期待看到什么
- 出现偏差时，顺手说一句“这里不对”
- 如果有两个相似按钮、列表项或区域，尽量用鼠标指给我看

不用担心说得不够像文档。UI Commander 原本就是为了把人类自然的指指点点和口头说明，变成后续还能继续利用的结构化线索。

## 录完之后会发生什么

一次录制结束后，事情并不会停在“留下了一段录屏”。

UI Commander 会把这次会话整理成一组可继续使用的材料，例如交互时间线、关注区域、截图、日志、转写结果和会话摘要。然后，当前对话里的 agent 会基于这些材料继续往前走:

- 理解你真正想修的点到底是什么
- 判断问题更像交互错误、状态同步问题，还是接口异常
- 必要时直接在当前仓库里改代码
- 或者先把分析结果讲清楚，再和你确认下一步

也就是说，UI Commander 不是“录制工具”本身，而是一座桥。它把“我来给你演示一下”这件事，接到“现在请你继续分析和修复”这件事上。

## 使用这个 skill 时，最重要的一件事

把它当成一种更自然的表达方式，而不是一个需要背说明书的工具。

你不必记住太多术语，也不必先把问题整理得很完整。只要在对话里明确想用 UI Commander，然后把问题在真实页面里演出来，就已经足够好了。

剩下的流程，本来就应该由 skill 和 agent 替你接住。

## 最后一段自我介绍

如果说普通描述是在“讲述一个 bug”，那 UI Commander 更像是在“把 bug 带到现场”。

我想做的，是让前端问题的沟通少一点转述，多一点亲眼所见；少一点猜测，多一点上下文；少一点“你应该懂我意思吧”，多一点“好，我已经看到它是怎么发生的了”。
